# Contributing Guide

Thank you for your interest in improving the Open AI Transformation Maturity Model.

## What We Welcome

- Public case studies mapped to maturity levels
- Improvements to assessment questions
- Clarifications of definitions
- Domain adaptations (e.g., telecom, finance, healthcare)
- Tooling and automation
- Visualizations and diagrams
- Corrections backed by evidence

## Contribution Standards

Please:

- Base claims on publicly verifiable sources where possible
- Distinguish evidence from opinion
- Avoid marketing language
- Do not overclaim Level 5 adoption
- Maintain backward compatibility where feasible

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make changes with clear documentation
4. Submit a pull request
5. Respond to review feedback

## Large Changes

For major structural changes, open an issue first to discuss.
