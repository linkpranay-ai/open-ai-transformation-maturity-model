# Glossary

> License: CC BY-SA 4.0

## AI Initiated
The very early phase where teams start using AI tools as helpers — things like code suggestions or quick explanations — but the overall development process stays basically the same.

## Augmented Coding
At this point, AI is doing real hands-on work (tests, boilerplate, refactors, etc.), but humans still review things closely before anything ships.

## Managed Agents
Here AI systems can take on multi-step tasks end-to-end, not just small snippets. People stop checking every line and focus more on whether the outcome is correct.

## Spec-Driven Development
Instead of writing code first, teams spend more effort describing what needs to happen — requirements, constraints, and success criteria. Implementation becomes something the machine largely handles.

## Autonomous Delivery / Dark Factory
In tightly scoped areas, software can move from request to release with very little human intervention. People set guardrails and watch the system rather than building each change manually.

## Intent Artifact
A structured description of what the system is supposed to do — behavior, limits, and expected results — written in a way machines can actually use.

## Scenario Architecture
All the tests, simulations, and checks used to verify that the system behaves correctly, regardless of how the code was produced.

## Agent Harness
The surrounding setup that makes AI agents useful in practice: tools, permissions, context, monitoring, and controls. Without this, agents are mostly just demos.

## Brownfield Extractability
How feasible it is to pull meaningful structure out of an existing legacy system so it can be automated or modernized. Some old systems cooperate; others really don’t.

## Evaluation Bottleneck
A situation where generating solutions becomes fast and cheap, but proving they actually work becomes the slow part.
