# AI Transformation Maturity Assessment Framework

> **License:** CC BY-SA 4.0  
> https://creativecommons.org/licenses/by-sa/4.0/

This framework provides a structured method for assessing how far a software organization has progressed toward AI-native engineering.

It complements the **5-Level AI Transformation Maturity Model** by defining measurable criteria across multiple capability dimensions.

👉 [Back to the 5-Level Maturity Model](../5-level-maturity-model.md)

---

## How to Use This Framework

A typical assessment involves:

1. Reviewing the scoring model  
2. Evaluating capabilities across the seven pillars  
3. Applying level-specific checklists  
4. Verifying readiness using stage gates  
5. Interpreting results and identifying constraints  

Organizations may operate at different levels across subsystems; the framework evaluates the dominant operating mode.

## Important Notes

- Higher maturity is not inherently better in all contexts  
- Many organizations will operate across multiple levels simultaneously  
- Domain constraints (safety, regulation, complexity) limit achievable autonomy  
- Progress should be gated by evaluation capability, not ambition
